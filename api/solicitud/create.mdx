## Overview

Creates a new transport solicitud (request) in the system. The solicitud is created directly in **PENDIENTE_APROBACION** status.

<Note>
  Missions are **not** created at this stage. The approver will create missions later in the Missions module after approval.
</Note>

## Authentication

Requires valid authentication token.

## Request Body

</ParamField>

<ParamField body="idDepartamentoSolicitante" type="integer" required>

  ID of the approver profile (FK to Perfil). Must be different from requester.
</ParamField>

<ParamField body="idTipoPrioridadSolicitud" type="integer" required>
  ID of the priority type (FK to TipoPrioridadSolicitud)
</ParamField>

<ParamField body="idTipoServicioSolicitud" type="integer" required>
  ID of the service type (FK to TipoServicioSolicitud)
</ParamField>

<ParamField body="asunto" type="string" required>
  Subject of the solicitud (5-250 characters)
</ParamField>

<ParamField body="descripcion" type="string">
  Detailed description of the solicitud
</ParamField>

<ParamField body="justificacion" type="string">
  Justification for the transport request
</ParamField>

<ParamField body="cantidadPasajeros" type="integer" required>
  Number of passengers (minimum 0)
</ParamField>

<ParamField body="nombresPasajeros" type="string">
  Names of passengers

  Date when service is required (YYYY-MM-DD format, cannot be in the past)
</ParamField>

<ParamField body="horaServicioRequerido" type="time" required>
  Time when service is required (HH:MM format)
</ParamField>

  List of locations (minimum 1 required)
  
  <Expandable title="Location Object Properties">
    <ParamField body="idLugar" type="integer" required>
      ID of the location (FK to Lugares)
    </ParamField>
    
    <ParamField body="esOrigen" type="boolean" required>
      True if this is the origin point
    </ParamField>
    
    <ParamField body="orden" type="integer" required>
      Order of the stop (0=origin, 1=first stop, etc.). Must be consecutive starting from 0.
    </ParamField>
    
    <ParamField body="descripcion" type="string">
      Description of what to do at this location (max 500 characters)
    </ParamField>
    
    <ParamField body="horaEstimadaLlegada" type="time">
      Estimated arrival time (HH:MM format)
    </ParamField>
    
    <ParamField body="requiereRetorno" type="boolean" default={false}>
      True if return trip is required from this location. Origin cannot require return.
    </ParamField>
    
    <ParamField body="horaEstimadaRetorno" type="time">
      Estimated return time (HH:MM format). Required if requiereRetorno=true. Must be after horaEstimadaLlegada.
    </ParamField>
  </Expandable>
</ParamField>

## Business Rules

- The solicitud code is auto-generated: `SOL-{YEAR}-{MONTH}-{CORRELATIVE}`
- The requester and approver must be different people
- Service date cannot be in the past
- At least one location must be provided
- Only one location can be marked as origin (esOrigen=true)
- Origin location cannot require return (requiereRetorno must be false)
- If requiereRetorno=true, horaEstimadaRetorno is mandatory
- Location order values must be unique and consecutive starting from 0
- Email notifications are sent to:
  - Requester: Confirmation email
  - Approver: Pending approval notification

## Status Codes

<ResponseField name="201" type="Created">
  Solicitud created successfully
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Validation error (invalid fields, requester = approver, invalid location data)
</ResponseField>

<ResponseField name="409" type="Conflict">
  Conflict error (duplicate, business rule violation)
</ResponseField>

<ResponseField name="422" type="Unprocessable Entity">
  Business logic validation error (date in past, invalid location rules)
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  Server error
</ResponseField>
