---
title: "Update Solicitud"
api: "PUT /solicitud/{id}"
description: "Update an existing solicitud"
---

## Overview

Updates an existing solicitud. This operation is only allowed for solicitudes in **PENDIENTE_APROBACION** status.

<Warning>
  Once a solicitud is approved, rejected, or in any other status, it cannot be updated.
</Warning>

## Authentication

Requires valid authentication token.

## Path Parameters

<ParamField path="id_solicitud" type="integer" required>
  The unique ID of the solicitud to update
</ParamField>

## Request Body

All fields are optional. Only the fields provided will be updated.

<ParamField body="idUsuarioAprobador" type="integer">
  ID of the approver profile (FK to Perfil)
</ParamField>

<ParamField body="idTipoPrioridadSolicitud" type="integer">
  ID of the priority type (FK to TipoPrioridadSolicitud)
</ParamField>

<ParamField body="idTipoServicioSolicitud" type="integer">
  ID of the service type (FK to TipoServicioSolicitud)
</ParamField>

<ParamField body="asunto" type="string">
  Subject of the solicitud (5-250 characters)
</ParamField>

<ParamField body="descripcion" type="string">
  Detailed description
</ParamField>

<ParamField body="justificacion" type="string">
  Justification for the request
</ParamField>

<ParamField body="cantidadPasajeros" type="integer">
  Number of passengers (minimum 0)
</ParamField>

<ParamField body="nombresPasajeros" type="string">
  Names of passengers
</ParamField>

<ParamField body="fechaServicioRequerido" type="date">
  Date when service is required (YYYY-MM-DD format, cannot be in the past)
</ParamField>

<ParamField body="horaServicioRequerido" type="time">
  Time when service is required (HH:MM format)
</ParamField>

<ParamField body="observaciones" type="string">
  Additional observations
</ParamField>

<ParamField body="lugares" type="array">
  New list of locations (replaces existing locations completely)
  
  <Expandable title="Location Object Properties">
    <ParamField body="idLugar" type="integer" required>
      ID of the location (FK to Lugares)
    </ParamField>
    
    <ParamField body="esOrigen" type="boolean" required>
      True if this is the origin point
    </ParamField>
    
    <ParamField body="orden" type="integer" required>
      Order of the stop (must be consecutive starting from 0)
    </ParamField>
    
    <ParamField body="descripcion" type="string">
      Description of activities at this location
    </ParamField>
    
    <ParamField body="horaEstimadaLlegada" type="time">
      Estimated arrival time (HH:MM format)
    </ParamField>
    
    <ParamField body="requiereRetorno" type="boolean" default={false}>
      True if return trip is required. Origin cannot require return.
    </ParamField>
    
    <ParamField body="horaEstimadaRetorno" type="time">
      Estimated return time (required if requiereRetorno=true)
    </ParamField>
  </Expandable>
</ParamField>

## Business Rules

- Only solicitudes in **PENDIENTE_APROBACION** status can be updated
- Service date cannot be in the past
- If `lugares` array is provided, it completely replaces existing locations
- All location validation rules from creation apply (see [Create Solicitud](/api/solicitud/create))

## Response

<ResponseField name="success" type="boolean">
  Indicates if the operation was successful
</ResponseField>

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="data" type="object">
  The updated solicitud object with all fields, including:
  - Updated solicitud details
  - Current lugares array
  - Associated misiones array
  - Requester and approver information
  - Current status details
</ResponseField>

## Status Codes

<ResponseField name="200" type="OK">
  Solicitud updated successfully
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Validation error (invalid fields, date in past)
</ResponseField>

<ResponseField name="404" type="Not Found">
  Solicitud with the specified ID does not exist
</ResponseField>

<ResponseField name="409" type="Conflict">
  Solicitud is not in PENDIENTE_APROBACION status
</ResponseField>

<ResponseField name="422" type="Unprocessable Entity">
  Business logic validation error (invalid location rules)
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  Server error
</ResponseField>
