---
title: "List Solicitudes"
api: "GET /solicitud"
description: "List solicitudes with pagination and filtering"
---

## Overview

Retrieves a paginated list of solicitudes with support for multiple filters. This endpoint also supports an unpaginated version at `/solicitud/todos/lista`.

## Authentication

Requires valid authentication token.

## Query Parameters

### Pagination

<ParamField query="pagina" type="integer" default={1}>
  Page number (minimum 1)
</ParamField>

<ParamField query="por_pagina" type="integer" default={10}>
  Records per page (minimum 1, maximum 100)
</ParamField>

### Filters

<ParamField query="idEstadoSolicitud" type="integer">
  Filter by status ID (exact match)
</ParamField>

<ParamField query="codigoEstado" type="string">
  Filter by status code (e.g., PENDIENTE_APROBACION, APROBADA, RECHAZADA, CANCELADA, COMPLETADA)
</ParamField>

<ParamField query="idDepartamento" type="integer">
  Filter by requesting department ID
</ParamField>

<ParamField query="idUsuarioSolicitante" type="integer">
  Filter by requester user ID
</ParamField>

<ParamField query="idUsuarioAprobador" type="integer">
  Filter by approver user ID
</ParamField>

<ParamField query="idTipoServicio" type="integer">
  Filter by service type ID
</ParamField>

<ParamField query="idTipoPrioridad" type="integer">
  Filter by priority type ID
</ParamField>

<ParamField query="fechaDesde" type="date">
  Filter by service date from (YYYY-MM-DD format)
</ParamField>

<ParamField query="fechaHasta" type="date">
  Filter by service date to (YYYY-MM-DD format)
</ParamField>

<ParamField query="busqueda" type="string">
  Search in solicitud code, subject, and description (partial match)
</ParamField>

## Unpaginated Endpoint

**GET /solicitud/todos/lista**

Returns all active solicitudes without pagination. Accepts the same filter parameters as the paginated endpoint (excluding `pagina` and `por_pagina`).

## Response

<ResponseField name="success" type="boolean">
  Indicates if the operation was successful
</ResponseField>

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="data" type="object">
  Pagination and results data
  
  <Expandable title="Data Properties">
    <ResponseField name="total" type="integer">
      Total number of records matching the filters
    </ResponseField>
    
    <ResponseField name="pagina" type="integer">
      Current page number
    </ResponseField>
    
    <ResponseField name="porPagina" type="integer">
      Records per page
    </ResponseField>
    
    <ResponseField name="registros" type="array">
      Array of solicitud objects. Each object includes:
      - All solicitud fields (Id, CodigoSolicitud, Asunto, etc.)
      - Requester details (NombreSolicitante, EmailSolicitante)
      - Approver details (NombreAprobador, EmailAprobador)
      - Department name (NombreDepartamento)
      - Priority name (NombrePrioridad)
      - Service type name (NombreTipoServicio)
      - Current status (CodigoEstadoActual, NombreEstadoActual, ColorEstadoActual)
      - Array of lugares (locations with full details)
      - Timestamps (FechaHoraCreacion, FechaHoraActualizacion)
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  For the unpaginated endpoint `/solicitud/todos/lista`, the response data is an array of solicitudes directly (no pagination metadata).
</Note>

## Status Codes

<ResponseField name="200" type="OK">
  Solicitudes retrieved successfully
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  Server error
</ResponseField>

<RequestExample>

```bash cURL (Paginated)
curl -X GET "https://api.example.com/solicitud?pagina=1&por_pagina=20&codigoEstado=PENDIENTE_APROBACION&idDepartamento=2" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```bash cURL (Search)
curl -X GET "https://api.example.com/solicitud?busqueda=transporte&fechaDesde=2026-03-01&fechaHasta=2026-03-31" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```bash cURL (Unpaginated)
curl -X GET "https://api.example.com/solicitud/todos/lista?codigoEstado=APROBADA" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```python Python
import requests

url = "https://api.example.com/solicitud"
headers = {"Authorization": "Bearer YOUR_TOKEN"}
params = {
    "pagina": 1,
    "por_pagina": 20,
    "codigoEstado": "PENDIENTE_APROBACION",
    "idDepartamento": 2
}

response = requests.get(url, headers=headers, params=params)
print(response.json())
```

```javascript JavaScript
const params = new URLSearchParams({
  pagina: 1,
  por_pagina: 20,
  codigoEstado: 'PENDIENTE_APROBACION',
  idDepartamento: 2
});

const response = await fetch(`https://api.example.com/solicitud?${params}`, {
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});

const data = await response.json();
console.log(data);
```

## Filtering Examples

### Filter by Status
```bash
GET /solicitud?codigoEstado=PENDIENTE_APROBACION
```

### Filter by Department and Date Range
```bash
GET /solicitud?idDepartamento=2&fechaDesde=2026-03-01&fechaHasta=2026-03-31
```

### Search and Filter by Priority
```bash
GET /solicitud?busqueda=transporte&idTipoPrioridad=1
```

### Get All Approved Solicitudes (Unpaginated)
```bash
GET /solicitud/todos/lista?codigoEstado=APROBADA
```
